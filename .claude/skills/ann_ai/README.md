# Ann_AI - AI Prompt Optimizer

使用结构化4-D方法将粗糙输入转换为精准优化的AI提示词

---

## 功能概述

Ann_AI 是一个专业的提示词优化工具，能够：
- 🔍 **智能分析**：自动解构用户输入，诊断清晰度、完整性、结构问题
- 🎯 **精准优化**：根据请求类型应用最优技术（CoT、Few-shot、多视角等）
- 🤖 **平台自适应**：自动判断最适合的AI平台（Claude/ChatGPT/Gemini）
- ⚡ **分级处理**：简单任务快速修复（BASIC），复杂任务深度优化（DETAIL）

---

## 快速开始

### 基础使用

直接输入你的粗糙提示词或想法：

```
帮我写一份项目报告
```

Ann_AI 会自动：
1. 判断复杂度（BASIC/DETAIL）
2. 选择最佳AI平台
3. 应用优化技术
4. 返回可直接使用的优化提示词

### 指定平台（可选）

如果你想使用特定AI平台：

```
用Claude优化：帮我分析这篇长文章的论证逻辑
```

### 指定模式（可选）

如果你想跳过自动判断：

```
DETAIL模式：设计一个在线课程的教学大纲
```

## 使用示例

### 示例 1：简单优化（BASIC模式）

**输入**：
```
写个周报
```

**输出**：
```
**Your Optimized Prompt:**
你是一位专业的项目管理顾问。请撰写一份本周工作周报：

本周完成的工作：
- [任务1]：具体成果和数据
- [任务2]：具体成果和数据

遇到的问题及解决方案：
- 问题：[描述]
- 解决方案：[描述]

下周计划：
- [任务1]：预期目标
- [任务2]：预期目标

要求：
- 使用要点列表，简洁明了
- 每项包含量化指标（如完成度、时间）
- 总长度约300字

**What Changed:**
添加了专业角色，提供了三段式结构（完成/问题/计划），明确了量化要求和格式规范
```

### 示例 2：复杂优化（DETAIL模式）

**输入**：
```
我要写技术博客
```

**处理**：
- 检测到"技术"关键词 → 触发DETAIL模式
- 提问：博客主题？目标读者？期望风格？
- 收集回答后生成完整优化结果（参见 EXAMPLES.md 案例2）

## 注意事项

### ✅ 最佳实践

- 直接描述你的需求，无需刻意组织语言
- 如有特定平台偏好，在输入中说明
- 复杂任务建议提供更多背景信息

### ⚠️ 限制说明

- 不会保存任何会话信息到记忆（memory）
- 自动判断可能不总是完美，可手动指定平台/模式
- 优化后的提示词需要根据实际情况调整参数占位符

### 🎯 质量标准

优化后的提示词保证：
1. 意图清晰无歧义
2. 参数具体可验证
3. 结构逻辑清晰
4. 可直接执行使用
5. 符合目标平台特性

## 高级功能

### 4-D 方法论

1. **DECONSTRUCT**：提取意图、实体、上下文、要求、约束
2. **DIAGNOSE**：审计清晰度、特异性、完整性、复杂度
3. **DEVELOP**：选择技术、分配角色、增强上下文、实施结构
4. **DELIVER**：构建提示词、选择格式、提供指导

详细说明见 `REFERENCE_4D_METHOD.md`

### 优化技术库

- **Foundation**：角色分配、上下文分层、输出规格、任务分解
- **Advanced**：思维链、Few-shot学习、多视角分析、约束优化

### 平台特定策略

- **Claude**：长上下文、深度推理、XML结构
- **ChatGPT**：结构化对话、多轮交互、清晰分段
- **Gemini**：创意任务、对比分析、探索性思维

完整案例见 `EXAMPLES.md`
