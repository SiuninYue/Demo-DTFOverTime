# 4-D METHODOLOGY REFERENCE

本文档提供完整的4-D方法论、优化技术库、平台策略和操作模式规范，供ann_ai skill在处理提示词优化时查阅。

---

## THE 4-D METHODOLOGY

### 1. DECONSTRUCT（解构）

**目标**：将用户输入分解为可分析的核心组件

**操作步骤**：
- 提取核心意图（用户真正想要什么）
- 识别关键实体（主题、对象、角色）
- 捕获上下文信息（场景、背景、限制）
- 明确输出要求（格式、长度、风格）
- 识别约束条件（禁止内容、必需元素）

**输出**：结构化的意图地图

**示例**：
- 输入："帮我写报告"
- 解构结果：
  - 核心意图：生成报告内容
  - 关键实体：报告（类型未知）
  - 上下文：缺失
  - 输出要求：缺失
  - 约束：无

---

### 2. DIAGNOSE（诊断）

**目标**：识别当前提示词的问题和改进空间

**检查维度**：

**A. 清晰度审计**
- 是否存在模糊表达（"好的"、"专业的"）
- 是否有多义词或歧义点
- 意图是否可被直接理解

**B. 特异性检查**
- 参数是否具体（"短"→"200字"）
- 要求是否可量化/可验证
- 是否提供足够的限定词

**C. 完整性评估**
- 缺失的关键信息（受众、目标、场景）
- 未定义的输出格式
- 未说明的质量标准

**D. 结构与复杂度**
- 是否需要分步骤执行
- 是否需要多轮交互
- 是否涉及复杂推理

**输出**：问题清单 + 复杂度等级（Simple/Moderate/Complex）

---

### 3. DEVELOP（开发）

**目标**：应用优化技术构建高质量提示词

**3.1 技术选择矩阵**

根据请求类型选择技术组合：

| 请求类型 | 推荐技术 | 原因 |
|---------|---------|------|
| **Creative（创意）** | 多视角分析 + 语气强化 | 激发多样性思维 |
| **Technical（技术）** | 约束优化 + 精确性聚焦 | 确保准确性和规范性 |
| **Educational（教育）** | Few-shot示例 + 清晰结构 | 降低理解门槛 |
| **Complex（复杂）** | 思维链(CoT) + 系统化框架 | 分解任务，提高推理质量 |

**3.2 角色分配**
- 为AI分配具体专业身份（"资深数据科学家" vs "AI助手"）
- 匹配角色与任务需求

**3.3 上下文增强**
- 补充缺失的背景信息
- 提供必要的参考框架

**3.4 逻辑结构实施**
- 应用清晰的分段结构
- 使用编号列表/要点
- 明确输入→处理→输出流程

---

### 4. DELIVER（交付）

**目标**：格式化并交付优化后的提示词

**4.1 构建优化提示词**
- 整合所有改进点
- 应用平台特定格式
- 确保可直接使用

**4.2 格式选择**
- 简单请求：简洁格式
- 复杂请求：完整格式（含解释）

**4.3 实施指导**
- 提供使用建议（如需要）
- 说明预期效果

## OPTIMIZATION TECHNIQUES LIBRARY

### Foundation Techniques（基础技术）

**1. Role Assignment（角色分配）**
```
格式：你是一位[具体专业身份]，擅长[核心技能]...
示例：你是一位拥有10年经验的Python后端工程师，擅长系统架构设计和性能优化...
```

**2. Context Layering（上下文分层）**
```
结构：
- 背景信息（Background）
- 当前状态（Current Situation）
- 目标期望（Desired Outcome）
```

**3. Output Specifications（输出规格）**
```
必需元素：
- 格式：[Markdown/JSON/纯文本/代码]
- 长度：[具体字数/页数]
- 结构：[章节划分/要点列表]
- 风格：[正式/友好/技术/创意]
```

**4. Task Decomposition（任务分解）**
```
格式：请按以下步骤完成：
1. [子任务1]
2. [子任务2]
3. [子任务3]
```

---

### Advanced Techniques（高级技术）

**1. Chain-of-Thought (CoT)（思维链）**
```
触发语：让我们一步步思考...
适用场景：复杂推理、数学问题、逻辑分析
示例：请一步步分析以下问题：
1. 首先识别关键变量
2. 然后建立变量之间的关系
3. 最后推导出结论
```

**2. Few-Shot Learning（少样本学习）**
```
格式：
示例1：
输入：[示例输入]
输出：[示例输出]

示例2：
输入：[示例输入]
输出：[示例输出]

现在处理：
输入：[实际输入]
输出：
```

**3. Multi-Perspective Analysis（多视角分析）**
```
格式：请从以下3个角度分析：
1. [角度1，如：技术可行性]
2. [角度2，如：成本效益]
3. [角度3，如：用户体验]
```

**4. Constraint Optimization（约束优化）**
```
格式：
必须满足：[硬约束]
应当优先：[软约束]
禁止使用：[排除项]
```

## PLATFORM-SPECIFIC STRATEGIES

### ChatGPT / GPT-4

**特性**：
- 擅长结构化对话
- 支持conversation starters
- 适合多轮交互任务

**优化策略**：
- 使用清晰的分段结构
- 提供明确的输出格式
- 利用System Message定义角色
- 适合需要连续对话的场景

**格式建议**：
```
System: 你是[角色]...
User:
任务：[任务描述]
要求：
- [要求1]
- [要求2]
格式：[输出格式]
```

---

### Claude

**特性**：
- 支持更长上下文（100K+ tokens）
- 擅长深度推理和分析
- 适合处理复杂文档

**优化策略**：
- 可提供更详细的背景信息
- 使用推理框架（如思维链）
- 适合长文本处理和深度分析任务
- 利用XML标签组织复杂内容

**格式建议**：
```
<context>
[详细背景信息]
</context>

<task>
[任务描述]
</task>

<requirements>
[详细要求列表]
</requirements>
```

---

### Gemini

**特性**：
- 擅长创意任务
- 支持对比分析
- 多模态能力强

**优化策略**：
- 强调创意和多样性
- 使用对比性提示
- 适合图像/文本混合任务
- 鼓励探索性思维

**格式建议**：
```
请创造性地完成以下任务：
[任务描述]

期望风格：[创意/探索性/多样化]
参考方向：[提供灵感但不限制]
```

---

### Others（通用平台）

**策略**：
- 应用普适的最佳实践
- 使用Foundation Techniques
- 避免平台特定语法
- 确保可移植性

## OPERATING MODES

### 模式判断规则

**BASIC MODE 触发条件**：
- 单一任务（无需分解）
- 通用场景（非专业领域）
- 明确输出要求
- 低复杂度（无需深度推理）

**DETAIL MODE 触发条件**：
- 多步骤流程
- 专业领域（技术/医疗/法律等）
- 复杂约束条件
- 高创意性要求
- 用户输入极度模糊

---

### DETAIL MODE 协议

**执行流程**：
1. 通知用户进入DETAIL模式
2. 提出2-3个针对性问题：
   - 目标受众是谁？
   - 期望的输出风格/格式？
   - 有哪些特定要求或限制？
3. 收集回答后执行完整4-D流程
4. 提供comprehensive优化结果

**适用场景**：
- 专业文档（技术报告、研究论文）
- 复杂创意项目（剧本、小说大纲）
- 系统化流程（项目规划、课程设计）

---

### BASIC MODE 协议

**执行流程**：
1. 快速诊断主要问题
2. 应用核心优化技术（Foundation Techniques）
3. 直接交付优化结果

**适用场景**：
- 日常写作（邮件、报告、总结）
- 简单查询（信息检索、翻译）
- 快速修复（明显的表达问题）

## RESPONSE FORMATS

### Simple Request Format（简单请求格式）

```
**Your Optimized Prompt:**
[优化后的完整提示词]

**What Changed:**
[简明扼要的改进说明，1-2句话]
```

**使用场景**：
- BASIC模式
- 改进点少于3个
- 用户需要快速结果

---

### Complex Request Format（复杂请求格式）

```
**Your Optimized Prompt:**
[优化后的完整提示词]

**Key Improvements:**
• [主要改进点1及其效果]
• [主要改进点2及其效果]
• [主要改进点3及其效果]

**Techniques Applied:**
[应用的优化技术，简要提及]

**Pro Tip:**
[使用建议或预期效果说明]
```

**使用场景**：
- DETAIL模式
- 改进点≥3个
- 涉及高级技术
- 需要实施指导

## QUALITY STANDARDS

### 优化后提示词必须满足：

1. **清晰性**：意图无歧义，任何人都能理解
2. **特异性**：参数具体，要求可验证
3. **完整性**：包含所有必需信息
4. **可执行性**：AI能直接按指令执行
5. **结构化**：逻辑清晰，易于阅读

### 检查清单：

- [ ] 是否分配了合适的角色？
- [ ] 是否提供了足够的上下文？
- [ ] 输出规格是否明确？
- [ ] 是否应用了适当的优化技术？
- [ ] 格式是否符合平台特性？
- [ ] 是否避免了模糊表达？

## NOTES

- **记忆限制**：不保存任何会话信息到memory
- **自动判断**：默认自动选择平台和模式
- **用户覆盖**：如用户指定平台/模式，遵循用户选择
- **效果优先**：在保持原意的前提下，优先提升实际效果
