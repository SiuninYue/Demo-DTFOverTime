# skills_mother 使用示例

本文档提供详细的使用案例，展示如何通过对话创建不同类型的Skills。

---

## 示例 1：创建简单的CSV分析Skill（基础场景）

### 需求描述
用户想要创建一个Skill，用于分析CSV文件并生成统计报告。

### 完整对话流程

**用户：** 帮我创建一个skill，用来分析CSV数据

**助手：**
```
我会通过几个问题帮你快速创建一个标准的 Skill。

Q1：这个 Skill 的名称是什么？（建议用小写+下划线，如 csv_analyzer）
Q2：它要解决什么问题？用一句话描述核心功能。
Q3：什么时候会用到它？触发场景是什么？
```

**用户：**
```
1. csv_analyzer
2. 自动分析CSV文件，生成统计报告和可视化图表
3. 当我上传CSV文件需要快速了解数据特征时
```

**助手：**
```
Q4：这个 Skill 需要哪些输入？（文件/链接/文本/参数）
Q5：期望的输出是什么？（文档/代码/报告/可视化）
Q6：执行流程分几步？每步做什么？（请按顺序说明）
```

**用户：**
```
4. 输入：CSV文件
5. 输出：Markdown格式的分析报告，包含基本统计信息和可视化图表
6. 流程：
   - 第一步：读取CSV文件，验证格式
   - 第二步：计算基本统计信息（行数、列数、数据类型、缺失值）
   - 第三步：生成描述性统计（均值、中位数、标准差等）
   - 第四步：创建可视化图表（分布图、相关性热图）
   - 第五步：生成完整的Markdown报告
```

**助手：**
```
这个需求涉及数据处理，我有两个追加问题：

Q7：是否需要创建Python脚本来自动化数据分析？
Q8：能否提供2-3个具体的使用案例？
```

**用户：**
```
7. 是的，需要脚本
8. 示例：
   - 分析销售数据，了解季度趋势
   - 分析用户行为数据，找出关键指标
   - 分析问卷调查数据，生成报告
```

**助手：** [展示生成计划] → [用户确认] → [生成所有文件] → [预览展示] → [用户确认] → [写入文件]

### 生成的文件
- `SKILL.md`（核心技能定义）
- `scripts/csv_analyzer.py`（数据分析脚本）
- `EXAMPLES.md`（3个详细示例）
- `README.md`（使用说明）

### 触发测试
**用户输入：** "帮我分析这个CSV文件，生成一份报告"
**预期行为：** Claude自动调用csv_analyzer skill，执行完整的分析流程

## 示例 5：批量创建协作的多个Skills

### 需求描述
用户想创建一套完整的产品开发工作流，需要3个协作的Skills。

### 对话流程

**用户：** 我想创建一套产品开发流程的skills，从PRD到设计到开发

**助手：**
```
看起来你需要创建多个相关的 Skills。我可以帮你：

1️⃣ 一次性规划整个工作流
   - 列出所有需要的 Skills
   - 定义它们之间的协作关系
   - 创建 CLAUDE.md 主控文件串联流程

2️⃣ 逐个创建每个 Skill
   - 按优先级依次创建
   - 确保各 Skill 的接口一致
   - 配置文档传递机制

你想采用哪种方式？
```

**用户：** 采用方式1，一次性规划

**助手：**
```
好的！让我们先规划整个工作流。

请告诉我：
1. 需要哪些 Skills？（例如：PRD生成、UI设计、代码生成）
2. 它们的执行顺序是什么？
3. 每个阶段的输入和输出是什么？
```

**用户：**
```
1. 需要3个Skills：
   - prd_generator：生成产品需求文档
   - ui_designer：根据PRD生成UI设计方案
   - code_generator：根据设计方案生成代码

2. 顺序：PRD → UI设计 → 代码生成

3. 流转：
   - PRD生成 → 输出 product_prd.md
   - UI设计读取 product_prd.md → 输出 ui_design.md + 设计图
   - 代码生成读取 ui_design.md → 输出代码文件
```

**助手：** [逐个收集每个Skill的需求（精简版问答）] → [批量生成所有Skills] → [创建CLAUDE.md主控文件] → [批量预览] → [确认] → [统一写入]

### 生成的文件结构
```

├── CLAUDE.md（主控文件，定义工作流）
└── skills/
    ├── prd_generator/
    │   ├── SKILL.md
    │   ├── REFERENCE_PRD_STANDARD.md
    │   └── README.md
    ├── ui_designer/
    │   ├── SKILL.md
    │   ├── REFERENCE_DESIGN_SYSTEM.md
    │   ├── EXAMPLES.md
    │   └── README.md
    └── code_generator/
        ├── SKILL.md
        ├── scripts/code_scaffold.py
        └── README.md
```

### CLAUDE.md 内容示例
```markdown
# 产品开发工作流

你是一个专业的产品开发助手，掌握从需求到设计到开发的完整流程。

## 可用 Skills

1. **prd_generator**：生成产品需求文档（PRD）
   - 触发：生成PRD、写PRD、产品需求文档
   - 指令：`/prd`

2. **ui_designer**：生成UI设计方案
   - 触发：UI设计、界面设计、设计方案

1. 需求阶段：调用 prd_generator → 生成 product_prd.md
2. 设计阶段：读取 product_prd.md → 调用 ui_designer → 生成 ui_design.md
3. 开发阶段：读取 ui_design.md → 调用 code_generator → 生成代码文件

## 指令使用

- `/prd`：启动PRD生成流程
- `/design`：启动UI设计流程（需先完成PRD）
- `/code`：启动代码生成流程（需先完成设计）
```

## 常见问题

### Q1：为什么 Claude 没有自动调用我创建的 Skill？

**A：** 可能的原因：
1. **description 中的触发关键词不够明确**：检查YAML头部的description字段，确保包含了用户可能使用的关键词
2. **关键词过于宽泛**：避免使用"处理"、"生成"等过于宽泛的词，要具体化（如"CSV分析"而非"数据处理"）
3. **Skill未正确加载**：确认.claude/skills/目录下的Skill结构正确

**解决方法：** 告诉我需要优化触发关键词，我会帮你修改description字段。

### Q2：如何决定是否需要创建 REFERENCE 文件？

**A：** 以下情况需要创建REFERENCE文件：
- 参考资料超过1000字
- 包含复杂的API文档或技术规范
- 需要提供详细的设计标准或数据模型
- 内容会频繁更新，需要独立维护

**简单判断：** 如果参考内容可以在SKILL.md的[参考资料]模块中用2-3段话说清楚，就不需要独立的REFERENCE文件。

### Q3：什么时候需要创建 Python 脚本？

**A：** 以下情况需要创建脚本：
- 涉及确定性的数据转换（CSV转JSON、格式转换等）
- 需要调用外部API
- 需要复杂的文件处理逻辑
- 需要数学计算或数据分析

**不需要脚本的情况：** 主要是文档生成、对话引导、信息整理等任务。

### Q4：如何修改已有 Skill 的执行流程？

**A：** 告诉我：
1. 需要修改哪个Skill
2. 哪个步骤需要调整
3. 调整后的流程是什么

我会读取现有的SKILL.md，修改[执行流程]模块，并同步更新README.md中的说明。

### Q5：可以为一个 Skill 创建多个 REFERENCE 文件吗？

**A：** 可以！如果参考资料涵盖多个主题，建议创建多个REFERENCE文件：
- `REFERENCE_API.md`：API文档
- `REFERENCE_DESIGN.md`：设计规范
- `REFERENCE_DATA_MODEL.md`：数据模型

在SKILL.md中分别引用：
```markdown
[参考资料]
- API文档：<reference path="REFERENCE_API.md" />
- 设计规范：<reference path="REFERENCE_DESIGN.md" />
- 数据模型：<reference path="REFERENCE_DATA_MODEL.md" />
```
